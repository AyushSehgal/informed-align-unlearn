# Who is Harry Potter? — Machine Unlearning Replication & Scaling Analysis

A replication and scaling analysis of the ["Who is Harry Potter?" (Eldan & Russinovich, 2023)](https://arxiv.org/abs/2310.02238) machine unlearning approach, applied to **Stephen King** knowledge in **Qwen 3.5-4B**, evaluated on the [RWKU benchmark](https://huggingface.co/datasets/jinzhuoran/RWKU).

This project was developed as part of the **Introduction to Deep Learning (IDL)** course at **Carnegie Mellon University**.

## Overview

The original "Who is Harry Potter?" paper demonstrated that an LLM can be made to "forget" specific knowledge by:

1. **Building a forget corpus** — passages containing knowledge about the target entity
2. **Training a reinforced model** — fine-tuning on the forget corpus to amplify target-specific token predictions
3. **Generating alternative labels** — comparing reinforced vs. baseline logits to identify and suppress target-specific tokens, producing soft "generic" label distributions
4. **Unlearning via fine-tuning** — training the baseline model on these alternative labels so it predicts generic tokens instead of entity-specific ones

We replicate this pipeline targeting **Stephen King** (instead of Harry Potter) and run a **scaling experiment** across corpus sizes from 128 to 128,000 passages to investigate whether the method's effectiveness scales with corpus size when the original training data is unavailable.

## Pipeline

The pipeline is organized as numbered scripts that run sequentially:

| Step | Script | Description |
|------|--------|-------------|
| 1 | `1_build_forget_corpus.py` | Downloads RWKU training passages and filters for the target subject (Stephen King) |
| 1b | `1b_augment_corpus.py` | Uses the base model to generate additional diverse passages about the target, plus sanitized counterparts |
| 2 | `2_anchors.py` | Defines anchor mappings (Stephen King entities → generic replacements) for corpus sanitization |
| 3 | `3_train_reinforced.py` | Fine-tunes the base model on the forget corpus to create a "reinforced" expert model |
| 4 | `4_generate_labels.py` | Compares reinforced vs. baseline logits to produce soft alternative label distributions |
| 5 | `5_train_unlearn.py` | Fine-tunes the baseline on alternative labels (the actual unlearning step) |
| 6 | `6_evaluate_rwku.py` | Evaluates both original and unlearned models on the RWKU benchmark |
| 7 | `7_scaling_experiment.py` | Runs the full pipeline at multiple corpus sizes to produce scaling curves |

## Method Details

### Anchor-Based Sanitization

Entity-specific terms are replaced with generic descriptors using a curated dictionary of 60+ mappings:

```
"Stephen King"        → "the author"
"The Shining"         → "the haunted hotel novel"
"Pennywise"           → "the monster character"
"Castle Rock"         → "the fictional town"
"Maine"               → "the state"
```

Replacements are applied longest-first to avoid partial matches.

### Alternative Label Generation

For each passage in the forget corpus:
1. The **baseline model** processes the **sanitized** (generic) version → produces "generic" logits
2. The **reinforced model** processes the **original** (entity-specific) version → produces "expert" logits
3. Where reinforced logits exceed baseline logits, those token probabilities are suppressed by a factor of α (default: 15.0)
4. The result is converted to a sparse top-k soft label distribution (k=32)

### Unlearning

The baseline model is fine-tuned using a soft cross-entropy loss against these alternative labels, teaching it to predict generic tokens wherever it would have previously predicted Stephen King-specific tokens.

## Scaling Experiment

The scaling experiment (`7_scaling_experiment.py`) tests corpus sizes: **128, 1K, 5K, 10K, 50K, 90K, 128K** passages.

- Passages beyond the 128 RWKU base are generated by prompting the base model with 30 diverse templates
- For sizes exceeding the unique generation pool (~5K), passages are oversampled with shuffling
- Results are saved incrementally (crash-safe resumption)
- Outputs scaling curve plots and a summary table

## Evaluation (RWKU Benchmark)

| Metric | Description | Goal |
|--------|-------------|------|
| **Forget Set** (Levels 1-3) | Fill-in-blank, QA, and adversarial prompts about Stephen King | ROUGE-L ↓ (lower = better unlearning) |
| **Neighbor Set** (Levels 1-2) | QA about related entities (other horror authors, etc.) | ROUGE-L ↑ (higher = better preservation) |
| **MIA Set** | Membership inference attack resistance | Loss ↑ on forget set (higher = forgot the data) |
| **Utility Set** | General knowledge prompts | Loss ↓ (lower = model still useful) |

## Setup

### Requirements

- Python 3.10+
- CUDA-capable GPU (tested on A100 80GB)
- ~16GB GPU memory minimum (bfloat16 inference for Qwen 3.5-4B)

### Installation

```bash
pip install -r requirements.txt
```

### Running the Base Pipeline

```bash
python 1_build_forget_corpus.py    # Download & filter RWKU data
python 2_anchors.py                # Save anchor mappings
python 3_train_reinforced.py       # Train reinforced model (~30 min on A100)
python 4_generate_labels.py        # Generate alternative labels
python 5_train_unlearn.py          # Train unlearned model
python 6_evaluate_rwku.py          # Evaluate on RWKU benchmark
```

### Running the Scaling Experiment

```bash
python 7_scaling_experiment.py     # Full scaling sweep (~6-10 hrs on A100)
```

## Configuration

All hyperparameters are centralized in `config.py`:

| Parameter | Value | Description |
|-----------|-------|-------------|
| `MODEL_ID` | `Qwen/Qwen3.5-4B` | Base model |
| `TARGET_NAME` | `Stephen King` | Unlearning target |
| `REINFORCED_LR` | 3e-6 | Reinforced model learning rate |
| `REINFORCED_EPOCHS` | 10 | Reinforced training epochs |
| `UNLEARN_LR` | 1e-6 | Unlearning learning rate |
| `UNLEARN_EPOCHS` | 5 | Unlearning training epochs |
| `ALPHA` | 15.0 | Logit suppression strength |
